----------------------------------------

     SpawnChecker readme

----------------------------------------

プレイヤー周囲でMobが湧く可能性があるブロックに、マーカーを表示して可視化します。
スライムのスポーンするチャンクを、枠で囲み可視化します。マルチプレイで使うことも可能(*1)です。
スポーナーの、スポーン範囲など様々な情報を、可視化できる機能(*2)を提供します。

(*1) マルチのワールドのシード値を知っている必要あり。設定はファイルに書き込む。
(*2) スポーン範囲(立方体)/同種Mob湧き制限範囲(立方体)/スポーナー起動範囲(球)/スポーン可否(ブロック単位に表示)

===================================================================================================
●操作説明
===================================================================================================

 ---------------------------------------------------------------------------
 | - モード変更              :  Ctrl + 「↑または↓」
 | - モードオプション変更    :         「↑または↓」
 | - 水平チェック範囲変更    :         「＋または－」(テンキーの＋/－)
 | - 垂直チェック範囲変更    : Shift + 「＋または－」(テンキーの＋/－)
 | - マーカーの明るさ        :  Ctrl + 「＋または－」(テンキーの＋/－)
 | - スポーナー可視化一時変更: スポーナーをクリック
 ---------------------------------------------------------------------------

  Ctrlキーを押しながら、カーソルキーの上下で、モードを変更できます。
  Ctrlを押さずにカーソルキーの上下だけを押すと、モードオプションを変更できます。

  チェック範囲は、テンキーの + または - で変更できます。
  シフトキーを押していない場合は、水平方向、押している間は垂直方向の範囲変更です。
  最小値は、自分中心に ±1 範囲、最大値は、自分中心に ±32 範囲です。
  範囲を広げ過ぎると負荷が高くなるため、FPSが下がったり、まともに動かなくなる可能性があります。
  マシンスペックと相談して調整してください。

  マーカーの明るさも変更できます。
  変更するには、CTRLキーを押しながら テンキーの + または - を押してください。
  最小値は-5、最大値は+5です。
  ※チェック範囲のキーを変更した場合、こちらの変更キーも変わります。

  なお、上下キー、+/-キーは、マインクラフトの「キー設定」画面より、好きなキーに変更が可能です。
  ただし、Shiftキー、Ctrlキーは固定のキーで変更はできません。


  スポーナー可視化モードではないモードを選択中に、スポーナーブロックをクリックしてぶん殴ると、
  一時的にスポーナー可視化モードに切り替わり、スポーナーの情報の描画を開始します。
  もう一度ぶん殴って描画を中止すると、元のモードに戻ります。


===================================================================================================
●インストール方法
===================================================================================================
MinecraftForge もしくは FML の単体インストールを行います。

▼Minecraft Forge 同梱FML
 http://www.minecraftforge.net/

 サイトが重たい場合は、以下のURLを参照する手もあります。
 http://files.minecraftforge.net/ (軽量なダウンロード専用ページ)
 http://jenkins.minecraftforge.net/job/forge/ (ここも重い、ただし変更履歴などは追いやすい)
 https://github.com/MinecraftForge/MinecraftForge (GitHub)

 基本的にはMinecraftのバージョンに対応した Recommended バージョンの
 universal のZIPファイルをダウンロードすればOKです。

 大きな変更がされたりしていない限り、導入はjarにぶち込むだけで動作するはずですが、
 ForgeのサイトやJPWikiなどを参照し、正しい手順で導入してください。

▼FML単体
https://github.com/cpw/FML
 download から fml-universal で始まるZIPファイルをダウンロードします。
 バージョンが新しいほうが不具合が少ない可能性が高いですが、うまくいかないときは別バージョンも試してみてください。
 心配な場合はForge の Recommended バージョンに同梱されているものは、安定していると思われます。

 基本的にModLoaderで動くModは全て動作します。
 ですが、ModLoaderが既存を書き換えた処理に依存しているつくりのModは、
 Modの作者が汎用的な処理に改修しない限り動作しない可能性があります。

 META-INF フォルダーがZIP内に入ってるかもしれませんが、こちらはコピーしなくて大丈夫です。
 また、minecraft.jar 内の META-INF も消してください。残っていると、おそらくMinecraftが起動しません。

▽ModLoader(非推奨)
http://www.minecraftforum.net/topic/75440-
 SpawnChecker 142v1 現在は動作すると思いますが(10/27現在もModLoaderが未リリースのため不明)、
 不具合が修正されなかったり、Mod作成にあたり不便な点が多いため、
 開発に利用するメリットがないので、将来的にはサポートを打ち切るつもりでいます。


[[インストール手順]]

  1. MinecraftForge もしくは FML単体を導入します。
     例に漏れず META-INF フォルダーは消してください。

  2. ダウンロードしたZIPファイルを展開せずに、%appdata%\.minecraft\mods フォルダーに配置します。

  3. Minecraftを起動します。

     設定を変更する場合は、一度 Minecraft を起動しメインメニューを表示したあと
     Minecraftを終了し、生成された設定ファイルを編集してください。

  ※過去バージョンの当MODを導入していた場合、その ZIP ファイルは、忘れずに削除してください。


===================================================================================================
●モード説明
===================================================================================================

SpawnChecker には以下の3つのモードがあります。

 - Spawnable point checker
 - Slime chunk finder
 - Spawner Visualizer

それぞれのモードには、モードオプションがあり、モード毎の表示内容を変更可能です。

以下に各モードの概要を記載します。

-------------------------------------------------------
[Spawnable point checker] - スポーンする場所の可視化 -
-------------------------------------------------------
▼概要

スポーンチェッカーのメイン機能です。

モンスターがスポーンする可能性のあるブロックに、マーカーを描画します。
モードオプションには次の5つがあり、何通りかの組み合わせ(オプションセット)が用意されています。

 1. マーカーのみ
 2. ガイドライン表示
 3. 強制表示モード
 4. ガストマーカー表示
 5. スライムマーカー表示

スライム、ガストのマーカーは、ネザーか通常世界かで選択できたりできなかったりします。
察せ！

▼デフォルトのマーカー色と、それに対応するモンスターについて

 ▽透過緑
 (通常)   エンダーマン, クリーパー, スケルトン, ゾンビ  ※マーカー中心に3x3の広さがあればスパイダーも対象
 (ネザー) 表示されません

 ▽透過黄
 (通常)   クリーパー, スケルトン, ゾンビ  ※マーカー中心に3x3の広さがあればスパイダーも対象
 (ネザー) ゾンビピッグマン, マグマキューブ

 ▽透過青
 (通常)   スパイダー, スケルトンライダーのみ
 (ネザー) マグマキューブのみ

 ▽透過白
 (通常)   表示されません
 (ネザー) ガスト

 ▽中心からずれた位置に透過水色
 (通常)   スライム
 (ネザー) 表示されません


※スパイダーがPOPするには、3x3x1のスペースが必要で、
  中心の y+1 座標は透過ブロックか、ブロックがない場合にのみスポーンが可能です。

※スライム、マグマキューブのみのマーカーの表示は、最小サイズのスライム、マグマキューブの大きさで判定します。
  この2種は、スポーン判定時にランダムで大きさが決まります。
  なので、実際にモンスターがスポーンする際の判定で、ランダムに選択された初期の大きさが最小サイズでない場合は、
  スポーンするよマーカーが表示されている場所でも、スポーンしてこない事もあります。

※ネザーの橋(遺跡)が生成される際は、橋の上にブレイズのスポーン判定があります。(そしてガストは判定から除外されます)
  そのため、ネザーの橋に野良ブレイズが沸いていることがあります。
  しかし橋の生成は、常時発生する可能性があるわけではないため、マーカーの表示対象とはしていません。
  野良ブレイズを見かけたらラッキーと思ってください。出会えても良いことは特にないですが＞＜。
  なお、1.4.xで追加されたウィザースケルトンも野良ブレイズと同じ条件で生成されます。
  そのため、こちらのスポーン判定も表示対象ではありません。ちょっとレアMobすぎる気がします。

※スケライダーは、スパイダーのスポーン判定に成功後、いくつかの条件を元に、ランダムでライダーに置換されます。
  そのため、スパイダーがスポーンできる場所であれば、スポーンする可能性があります。
  ただし、場所によっては置換される確率0%のところもありそうなので、絶対にスポーンしない場所もあります。(ﾀﾌﾞﾝ)
  そのため、スポーン判定の対象とはしていません。

※ガストは浮いてますが、スポーン判定は他のモンスターと同様に、上面が平らな通常ブロックの上にしかありません。
  (1.3.xからは、逆さまに設置した階段やハーフロックの上も判定対象ですが…)
  ただし、ガストは大きいため、他のモンスターよりも大きなスペースがある場所にしかスポーンは出来ません。
  5x5x5ブロック分くらいの空間があればスポーンします。
  ガストのマーカーも他のマーカー同様、Mobの大きさで空間のチェックを行い、
  スペースが十分にある場所にのみ表示されるようになっています。

※通常世界ではガストを含むモードオプション、ネザーではスライムを含むモードオプションは、全て選択不可になります。
  また、通常世界でも、ワールドシードが不明な場合はスライムを含むモードオプションも選択不可になります。

※モードオプションのセットは設定ファイルより変更することが可能です。
  でも割とスパゲッティな設定なのでお勧めはしません。

※ジエンド…？ソンナノシリマセーン！
  暗ければヤツがでる。それだけです。
  「見えなくたって、わかるだろ…？」


-------------------------------------------------------
[Slime chunk finder] - スライムチャンクの可視化 -
-------------------------------------------------------
▼概要
スライムチャンク、スライムのスポーンマーカーのみに機能を絞ったモードです。
チェック範囲内に入ったことのあるスライムチャンクを囲む枠を描画します。

チェック範囲に入ったことのないスライムチャンクは表示されないため、
何も表示されない場合、一度チェック範囲を広げるか、周囲を歩き回ってみてください。
一度表示した位置はしばらくキャッシュされます。離れてるとそのうち忘れます。

マルチプレイでは、サーバーのシードを正しく設定してある場合のみ、表示が可能です。

通常のスライムマーカーとガイドライン、チャンクマーカーを組み合わせた、数パターンのモードオプションがあります。
通常のスポーンマーカーとガイドラインは、y40以下のブロックに、スポーン条件が整っている場合のみ表示されます。
チャンクマーカーはスライムの湧くチャンクに、自分のy座標を基点に表示されます。
地上からスライムチャンクを探すのに便利です。


-------------------------------------------------------
[Spawner Visualizer] - スポーナーの可視化 -
-------------------------------------------------------
▼概要
モンスタースポーナーの情報の可視化を行うモードです。
モンスターのスポーン範囲、スポーナーの起動する範囲を描画します。

  vvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvvv
   このモードは、近くにモンスタースポーナーがないと何も表示できません。
  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

モードを選択した状態でスポーナーに隣接し、Ctrlキーを押すか、
このモードに切り替えた状態で、スポーナーブロックを叩く、素手で右クリックすることで、
スポーナーの情報の描画を開始します。

また、他のモードにしている状態でも、スポーナーブロックを叩くか右クリックをする事で、
一時的にこのモードに切り替えることが可能です。(スポーナーから離れるかもっかい叩くと戻ります)

情報の表示内容は、上下キーでオプションを変更してください。

情報表示を開始した場合、スポーナーに赤い枠が表示されます。

非表示にしたい場合は、モードを切り替えるか、スポーナーをぶん殴ってみてください。おとなしくなります。

表示できる情報は、次のとおりです。
 1. スポーン判定の行われる範囲
 2. スポーン6匹制限の判定範囲
 3. 明るさとMobの大きさを考慮した、スポーン可否判定
 4. スポーナーが起動する範囲

オプション名は英語表記になっていますが、なんとなく雰囲気で察してください。


ここからは、あまり重要ではない説明です。
スポーン可否判定のマーカーは、座標の中心が選択されたことを想定してチェックしています。
しかし、スポーン判定が行われるXYZの空間座標は、「1. スポーン判定の行われる範囲」の枠内の
全ての場所からランダムに選択されるため、常にブロックの中心の座標が選ばれるわけではありません。
 ※つまり、小数点以下の値を含めた座標で、スポーン判定を行う座標の中心が決定されます。

そのため、このマーカーが「出現する」、または「出現しない」と示している場所であっても、
中心からずれた場所が選択された場合は出現できる、出来ないという事が起こります。

 ※例えば、スポーン可能範囲を全て鉄作で埋めてしまうと、マーカーはスポーンできない判定になりますが、
   実際は、鉄作と鉄作の隙間にモンスターはスポーン可能なため、暗くすればスポーンしてきます。

また、1.4.2でスポーン判定に使っている指定座標の明るさ情報が、特定条件下で取得できなくなっています。
(判定を行うサーバー側でのみ情報を持っているようです。おそらく光源バグの修正の一環での変更ではないかと考えています。)
そのため、上空にブロックがなく、空が見えている座標での明るさを、現在時刻などを条件にしてアバウトに決定しているため、
スポーン判定が間違う可能性があります。おそらく即補正されるレベルの短い時間ですが。

なので、表示されるマーカーはあくまでも明るさを判定するための参考情報として考えてください。


スポーナーの起動範囲(炎が点りスポーナーが動き始める範囲)はスポーナーを中心とした球体です。
球体の描画は、ポリゴンの頂点数が多いため、それなりに重たい処理になります。

また、球体の内から外、外から内を見た場合、向こう側に居る敵など、一部の情報が見えなくなったりします。
この表示を行う再は、周囲に十分注意してご利用ください。

※設定で、アクティブエリア球体の面の不透過率を0に設定し、色を透明にすると、向こう側も見えるようになります。


===================================================================================================
●設定ファイルについて
===================================================================================================
 ※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※
 ※※ 設定を変更する際は Minecraft を終了してから行ってください。※※
 ※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※※

  初回起動時に、以下のファイルが作成されます。

  □ MOD設定ファイル
    %appdata%\.minecraft\config\mod_SpawnChecker.cfg

     最後に選択したモード、各モードの最後に選択したモードオプション、モードオプションセット、
     チェック範囲、チェック間隔、マーカーの明るさ、各種マーカーの色、メッセージ表示時間と位置、
     レンダリング用エンティティの位置の設定、といった各種の設定値がここに保存されています。

  □ サーバーシード値設定ファイル
    %appdata%\.minecraft\config\mod_SpawnChecker\servers.txt

     サーバーに対応するワールドシード値を設定することができます。


  いずれもただのテキストファイルなので、任意のテキストエディタ(メモ帳など)で開いて編集してください。
  ただし、日本語や全角空白などを記入しないように注意してください。
  正しく動作しない原因になる可能性があります。
  なお、いずれのファイルも、編集を行う際は Minecraft は終了してから実施ください。

  ※ ModLoaderを使用している場合の注意 ※
  MOD設定ファイルの編集の際、checksum の項目の値は変更しないでください。
  この値は、ModLoader で設定されているデフォルト値の情報を中途半端に保持している項目です。
  なのでこの値を変更するか削除すると、設定がリセットされて、全てデフォルト値に戻ってしまいます。

  FMLでは問題ありません。


  ▼設定項目の説明
  -- -- -- -- -- -- -- -- -- -- -- -- -- -- -- -- --

  ●[有効化アイテムの指定]

    enabling_item_ids

      手に持った際、マーカー表示を有効化するためのアイテムであると判断するための、アイテムのIDを設定します。
      デフォルトは、松明のIDである 50 が指定されています。

      カンマ区切りで複数のアイテムID(またはブロックID)を指定することが可能です。

      ただし、色つき羊毛のような形式のIDは、IDの部分のみで指定してください。ダメージ部分での指定は出来ません。

      IDでの指定なので、modで追加したアイテムであっても設定が可能です。
      値は 0～31999 の範囲で指定してください。(それ以外は無視されます)

      例えば、明るさを持つアイテムのIDを指定したい場合、

       enablingPromotedItems=50,89,91

      となります。
      (グロウストーンは 89 、ランタンは 91 )

      IDはこちらを参照してください。
      http://www.minecraftwiki.net/wiki/Data_values

  ●[チェック間隔]

    checking_duration

      ミリ秒で、スポーン可否チェックなど、チェック処理の間隔の最小値を指定します。
      ここで指定した時間が経過したあと、次の画面更新の際にチェック結果が画面に反映されます。
      (つっても、0コンマ何秒とかの世界です)
      デフォルト値は 300 ミリ秒で、おおよそ1秒間に3回程度のチェックを行います。

      短いほど高頻度でチェックを行うため、
      トーチを置いたあと直ぐに明るさが反映されマーカーが消える、などの違いが出てきます。
      ただし、間隔が短いほどFPSの低下を招きます。
      変更する場合、快適なプレイが出来る範囲で設定することをオススメします。

  ●[メッセージが消えるまでの時間]

    information_timeout

      ミリ秒で指定してください。デフォルトは2000ミリ秒(2秒)です。
      この時間が経過したあとの画面更新から、情報メッセージの描画がされなくなります。

  ●[スポーナー可視化の右クリック使用可否]

    enabled_spawner_visualizer_right_click

      false にすると右クリックでの ON / OFF ができなくなります。
      SpawnerGUI のような、一部のModの機能を優先してこちらの機能を利用しないようにしたい場合に false にしてください。
      false に設定した場合でも、左クリックや、スポーナー可視化モード変更後にCtrlキーを押下して、表示をONにする事は可能です。
      クリエイティブモードでは、左クリックをするとブロックを破壊してしまうので、ケースバイケース。

  ●[色の変更]

    Rが赤、Gが緑、Bが青、Aが透過(アルファチャンネル)で、それぞれ0～255の数値を指定できます。
    RGBは数字が大きいほうが明るくなり、Aは0が完全な透明、255が不透明です。

    spawn_marker1_color_R/G/B/A (エンダーマンもPOP可マーカー色 デフォ緑色)
    spawn_marker2_color_R/G/B/A (ゾンビ/スケルトン/クリーパー/ゾンビピッグマンPOP可マーカー色 デフォ黄色)
    spawn_marker3_color_R/G/B/A (スパイダー/マグマキューブPOP可マーカー色 デフォ青)
    spawn_marker4_color_R/G/B/A (スライムマーカー色 デフォ水色)
    spawn_marker5_color_R/G/B/A (ガストマーカー色 デフォ白色)

      スポーンチェッカーのマーカーとガイドラインの色です。

  ●[メッセージの位置オフセット]
    info_v_offset
    info_h_offset
      メッセージの表示位置のオフセットが設定できます。
      垂直(v)オフセットは、0が画面の中央になり、-100が画面の上、100が画面の下になります。
      水平(h)オフセットは、デフォルトで画面左端より5ピクセルの位置に表示されるため、
      そこから左方向はマイナス、右方向はプラスで、値を指定できます。
      残念ながら、画面右側に表示するモードは現在のところありません。

  ●[エンティティの位置オフセット]
    entity_offset_x/y/z
      デフォルトは y + 5 で、プレイヤーの頭上5ブロック上に配置されます。
      このエンティティのレンダリングを行う際に、全てのマーカーを描画しています。
      追加エンティティが他のModと何らかの競合を起こす場合に、
      このエンティティ位置を移動することで、回避できる可能性があります。

  ※デフォルト値に戻したい項目は、その項目だけをファイルから削除すると、

  ※数値指定の項目には、設定できる範囲が制限されています。

 -デバッグ用設定メモ
  debug=true/false (default: false)
    デバッグレベルのログを出力するようになります。不具合調査時に役に立つことがあります。

  trace=true/false (default: -)
    通常設定項目自体がファイルに出力されていません。
    trueを設定した場合は、頻出するログが出力されるようになります。通常はこの設定は不要です。


●マルチプレイでのワールドシードの設定について

  Minecraft 1.2.3より、マルチプレイ時にワールドのシード値が取得できなくなりました。
  シードが不明な場合、スライムの判定が行えません。

  この仕様変更に対応するために、サーバーのシードを設定ファイルより読み込むことが出来る機能を設けています。

  %appdata%\.minecraft\config\mod_SpawnChecker\servers.txt ファイルに、
  サーバー名とポートと、そのサーバー情報に対応するシードを設定することで、
  マルチプレイ時にもスライムの判定が行えるようになります。

  以下の書式で設定してください。

    [host_name]/[port]=[seed]
    ホスト名またはIPアドレス、半角スラッシュ、ポート = ワールドシード(数値)

  ポート番号も「必須」です。
  Minecraft 内のサーバーの設定で、ポートを指定しなかった場合は、デフォルトのポート番号 25565 になります。

  シードは文字列ではなく64bitの数値(long型の値)で指定します。

  この値は、level.dat 内に格納されている情報です。詳細はWikiなどで検索してください。
  http://www26.atwiki.jp/minecraft/pages/73.html#CheckSeeds

  なお、1.3.xより、/seed コマンドにてシード値が取得できるようになりました。
  ただし、マルチプレイの環境では、OP権限(サーバー管理者権限)があるユーザーのみが使用可能です。
  (シングルプレイでは、全ユーザーが常に使用可能です。)

  例に漏れず、この設定も、変更する際は Minecraft を終了してから行ってください。

  なお、ゲーム内よりサーバーのシード値を取得する方法はないです。
  各サーバーの管理者にお問い合わせください。
  (必ず教えてもらえるかはわかりません。聞く場合は迷惑にならないようにね！)

  設定がうまく反映されていない場合は、FML、またはModLoader のログを確認してみてください。
  設定ファイルを検索した際のサーバーの情報が、以下のような形式でログファイルに出力されているので、間違いがないかを確認できます。
  (下記はFMLのログ)
  > 2012-08-20 06:48:46 [FINEST] [mod_SpawnChecker] primary key=localhost/25565
  > 2012-08-20 06:48:46 [FINEST] [mod_SpawnChecker] secoundary key=127.0.0.1/25565
  
  primaryは、サーバーのホスト名が取れた場合はホスト名、取れなかった場合はIPアドレスになります。
  secoundaryは、primaryで取得できなかった場合に使用される、サーバーのIPアドレスです。
